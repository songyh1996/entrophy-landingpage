User Story

As Jina, I want to save outfits I discover to my personal closet so I can buy items later when I'm ready.

Acceptance Criteria 2 of 4

Heart icon on post saves outfit to Closet tab
Closet shows Pinterest-style grid of saved outfits
Tapping saved outfit shows tagged items with prices
Buy button deep-links to retailer with affiliate tag

Edge Cases

Item goes out of stock after saving
Show "Sold out" badge on the saved card. Surface 3 visually similar items from the same retailer below the original.
Creator deletes original post
Keep saved item but mark "Original removed." Item still purchasable; remove the source post link.
Price changes after saving
Show current price as primary; show "was $89 when saved" as muted strikethrough next to it.
Click to add a block, or type / for commands
LS
This spec looks solid. One question — do we need a save limit?
M
No limit. Closet is the retention hook — more saves = more return visits. Storage cost is negligible.
LS
Makes sense. What about when a price changes after saving — show the old price or new?
M
Both. Show current price + "was $89 when saved." I've added this as an edge case to the spec.
